# Nginx 502/504 错误排查

适用场景：
- 浏览器或 curl 访问时返回 `502 Bad Gateway`
- 请求超时返回 `504 Gateway Timeout`
- Nginx 作为反向代理，后端服务无响应或响应太慢

常见原因：
- 后端应用（upstream）未启动或崩溃
- upstream 地址/端口配置错误
- 后端处理太慢，超过 Nginx 的 `proxy_read_timeout`
- upstream 连接数耗尽

步骤（最小可用）：

1) 查看 Nginx 错误日志
```bash
sudo tail -100 /var/log/nginx/error.log
# 或自定义日志路径
sudo tail -100 /etc/nginx/logs/error.log
```

2) 确认后端服务是否在运行
```bash
# 查看端口是否监听
ss -ltnp | grep :8000
# 或用 curl 直接测试后端
curl -v http://127.0.0.1:8000/api/health
```

3) 检查 Nginx upstream 配置
```bash
sudo nginx -T | grep -A5 "upstream\|proxy_pass"
# 确认 proxy_pass 中的地址和端口与后端一致
```

4) 检查 Nginx 配置语法并重载
```bash
sudo nginx -t
sudo systemctl reload nginx
```

5) 调整超时参数（针对 504）
```bash
# 在 nginx.conf 或 server/location 块中添加：
# proxy_connect_timeout 60s;
# proxy_send_timeout    120s;
# proxy_read_timeout    120s;
sudo vim /etc/nginx/nginx.conf
sudo systemctl reload nginx
```

6) 查看后端服务日志
```bash
# 以 uvicorn/gunicorn 为例
journalctl -u myapp.service -n 100
# 或直接查看进程日志文件
tail -100 /var/log/myapp/app.log
```

验证：
```bash
# 直接 curl 后端确认正常
curl -I http://127.0.0.1:8000/
# 通过 Nginx 访问确认不再 502/504
curl -I http://localhost/
```

回滚：
```bash
# 恢复 nginx.conf 修改前的备份
sudo cp /etc/nginx/nginx.conf.bak /etc/nginx/nginx.conf
sudo systemctl reload nginx
```

注意事项：
- 502 通常是后端未运行；504 通常是后端运行但响应太慢，排查思路不同
- 修改配置前先 `sudo cp nginx.conf nginx.conf.bak` 备份

关键词：
- nginx 502 bad gateway
- nginx 504 gateway timeout
- proxy_pass upstream
- nginx error log
- proxy_read_timeout
- backend service down
