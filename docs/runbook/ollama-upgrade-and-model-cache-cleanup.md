# Ollama 升级与模型缓存清理

适用场景：
- 需要升级 Ollama 版本
- 磁盘被模型缓存占满
- 升级后出现模型兼容、服务异常或缓存残留问题

症状：
- Ollama 版本过旧
- 模型目录过大
- 升级后服务无法正常启动

快速判断：

```bash
ollama -v
ollama list
df -h
```

修复步骤：

1. 升级前确认
- 当前版本
- 当前模型列表
- 模型缓存路径
- 是否有业务依赖正在使用

2. 升级
- 按当前安装方式执行官方升级流程
- 升级后重启服务

3. 检查模型缓存
- 列出已安装模型
- 删除不再需要的模型

```bash
ollama rm MODEL_NAME
```

4. 若磁盘仍然紧张
- 评估是否迁移模型目录到更大磁盘
- 清理废弃模型和重复副本

回滚：
- 恢复旧版本安装包或旧服务配置
- 若新版本不兼容，切回旧版本并验证模型可用性

验证：
- `ollama -v` 已更新
- `ollama list` 正常
- 最小模型推理成功
- 磁盘占用下降到可接受范围

风险提示：
- 不要直接手工删除模型目录文件，优先使用官方命令
- 升级前要明确回滚版本
- 模型缓存清理前确认是否仍被业务依赖

关键词：
- ollama upgrade
- ollama cache cleanup
- ollama rm
- ollama disk full
- ollama model cache
- ollama 升级
- 模型缓存清理
- ollama 磁盘
