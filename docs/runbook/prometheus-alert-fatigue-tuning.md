# Prometheus / Alertmanager 告警噪声治理

适用场景：
- 告警过多、过碎、重复、误报，值班团队出现告警疲劳
- 需要优化告警分级、聚合、静默与升级路径

症状：
- 一个问题触发几十条告警
- 值班群长期刷屏
- 大量低价值告警掩盖真正事故
- 告警触发频繁但恢复也很快

快速判断：
- 看过去 7 天告警总量
- 看最高频规则
- 看误报率和重复率
- 看告警是否按服务、实例、集群正确聚合

修复步骤：

1. 先分类告警
- 可行动告警
- 观察性告警
- 噪声告警
- 已过时告警

2. 优化规则
- 提高阈值或延长持续时间
- 为瞬时抖动增加 `for`
- 用聚合指标替代单实例噪声

3. 优化分组与路由
- 在 Alertmanager 中按服务、环境、严重级别聚合
- 为低优先级告警下沉通知渠道

4. 加入静默与抑制
- 例如主告警触发时抑制下游次级告警

5. 建立告警复盘
- 每周清理高噪声规则
- 对无人处理、无动作价值的规则直接下线

回滚：
- 若新规则过于宽松，恢复旧规则并重新评估阈值
- 调整路由时保留一个审慎窗口，防止误吞高优先级告警

验证：
- 告警数量下降
- 关键事故仍可被及时发现
- 值班响应质量提高

风险提示：
- 降噪不能靠简单关告警，要保留真正可行动告警
- 聚合和抑制配置错误可能隐藏真实事故
- 所有告警规则应有负责人
