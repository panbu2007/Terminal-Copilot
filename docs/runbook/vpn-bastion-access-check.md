# VPN 与跳板机访问检查

适用场景：
- 需要访问内网服务，但 VPN 或跳板机不可用
- 运维入口正常时灵时不灵
- 需要区分问题在本地网络、VPN 客户端、跳板机还是目标主机

症状：
- VPN 无法连接
- 跳板机 SSH 登录失败
- VPN 连上后仍无法访问内网资源
- 只有部分网段可达

快速判断：
- 本机网络是否正常
- VPN 是否真正建立路由
- 跳板机是否在线
- 目标主机是否允许来源 IP

修复步骤：

1. 检查本地网络
- 先确认不是本地断网、DNS 异常或代理冲突

2. 检查 VPN 状态
- 客户端是否已连接
- 路由是否注入成功
- DNS 是否切换到企业内网解析

3. 检查跳板机
- SSH 是否可达
- 账号、密钥、MFA 是否正常

4. 检查目标主机
- 是否仅允许内网或跳板机来源
- 防火墙或安全组是否已放通

5. 分层验证
- 先测到 VPN 网关
- 再测到跳板机
- 再测到内网服务

回滚：
- 恢复上一个稳定 VPN 配置文件或客户端版本
- 恢复旧的 bastion 入口配置

验证：
- VPN 路由和 DNS 生效
- 可 SSH 到跳板机
- 可从跳板机访问目标资源

风险提示：
- 不要为排障临时暴露内网服务到公网
- VPN 客户端升级和路由策略变更常导致大面积访问异常
- 跳板机应保留独立审计与权限边界

关键词：
- vpn access
- bastion host
- jump host
- internal network access
- vpn route
- ssh bastion
- 跳板机排查
- 内网访问
