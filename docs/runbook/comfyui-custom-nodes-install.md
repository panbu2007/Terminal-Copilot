# ComfyUI 自定义节点安装与排查

适用场景：
- 需要给 ComfyUI 安装自定义节点
- 安装后节点未显示、依赖报错、启动失败
- 需要降低安装未知节点带来的供应链风险

症状：
- 节点安装后 UI 中看不到
- 启动时报 Python 依赖错误
- 某个节点导致 ComfyUI 整体启动失败

快速判断：
- 看节点仓库 README
- 看是否需要额外 Python 包、系统包、模型文件
- 看 ComfyUI 启动日志

修复步骤：

1. 先确认来源可信
- 优先官方索引、维护者仓库、活跃项目
- 不要直接安装来源不明节点

2. 安装节点
- 复制到 `custom_nodes/`
- 按节点 README 安装依赖

3. 重启 ComfyUI
- 安装后通常需要重启

4. 若节点未显示
- 看启动日志
- 检查目录结构是否正确
- 检查依赖是否成功安装

5. 若某节点导致整体失败
- 先移出问题节点目录
- 再单独排查该节点依赖

回滚：
- 删除或移出问题节点
- 恢复到上一个稳定的节点集合

验证：
- 节点在 UI 中可见
- 最小工作流可运行

风险提示：
- 自定义节点是高风险扩展面
- 节点可能引入不受控依赖和额外下载脚本

关键词：
- comfyui custom nodes
- comfyui node install
- comfyui 节点
- custom_nodes
- 节点依赖错误
- comfyui 扩展
- 自定义节点安装
- comfyui 排障
