# MVP 用户流（评委 3 分钟）

## 核心理念
- 不是“词典补全”，而是**上下文触发的下一步工作流建议**。
- 每一步都有：命令 + 解释 + 风险级别 +（可选）执行 + 输出校验。

## 页面结构
- 左侧：Web 终端（输入/输出）
- 右侧：步骤面板（Plan → Execute → Verify）
- 顶部：模式切换
  - 只建议（不执行）
  - 建议 + 执行（需要确认）

## 交互闭环
1. 用户在终端执行/编辑
2. 系统观察上下文：最近命令、退出码、关键输出、文件变更事件（MVP 先做“命令输出触发”）
3. Planner 给出 1~3 条下一步候选
4. 用户一键接受（填入终端）或一键执行
5. Verifier 解析输出，标记成功/失败，失败给出分支修复建议
