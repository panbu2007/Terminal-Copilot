# 评委 3 分钟演示脚本（2026-01-25）

目标：用最少操作证明 3 件事：
1) 它不是“命令词典”，而是**可执行的下一步工作流**。
2) 每一步有**结构化解释**（why/risk/rollback/verify）+ 引用依据 + 安全护栏。
3) 能形成**任务流时间线**（回合追加、不覆盖）并可导出回放。

## 0. 开场（10s）
- 一句话：Terminal Copilot 让新人在命令行里“少走弯路”，把零散命令串成可验收工作流。
- 指给评委看：左边是真终端输入输出；右边是任务流时间线（每执行一次追加一个回合）。

## 1) Demo2：纠错（60s，推荐）
在终端输入：
- `git chekcout main`

讲解要点：
- 右侧出现建议卡：修复拼写到 `git checkout ...`，并显示 `agent=rules`（轻量多 Agent 分工可见）。
- 建议卡能看到 why/risk/verify（结构化字段）；并可能带引用依据。

点击：
- 「执行」（或「插入到终端」再回车）

验收点（现场观察）：
- 右侧时间线新增“回合 #1”，包含：执行命令、stdout/stderr 摘要、校验/护栏（若有）。
- 若失败（例如不在 git 仓库），下一步建议应出现自愈类建议（例如定位/帮助），并标记 `agent=selfheal`。

## 2) 终端体验加分（30s）
- 上下方向键：回看历史输入。
- Ctrl+C：在 local 执行中可 best-effort 中断（会调用 `/api/interrupt`）。
- 超长输出：终端会截断避免卡顿；右侧时间线提供 stdout/stderr 下载。

## 3) Demo3：跨平台端口检查（40s）
对评委说：这是典型新人问题“8000 被占用吗”。
- 输入：`端口 8000 被占用吗`（或 `?端口 8000 被占用吗` 只建议模式）

验收点：
- 根据平台返回不同命令（Windows: `netstat -ano | findstr :8000`）。
- 执行后若命中 verifier 规则，会在时间线“校验”区展示结论。

## 4) 证据链：导出回放（20s）
点击右上角：
- 「导出回放」

讲解要点：
- 导出的是 events.json，可作为评审材料：建议→执行→校验→中断等行为都能追溯。


## 5) 收尾（10s）
- 总结：建议可执行、结果可验证、过程可回放；面向新人最常见的“下一步不知道做啥/失败怎么自愈”。
