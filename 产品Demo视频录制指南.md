# Terminal Copilot — 产品 Demo 视频录制指南（1–3 分钟）

> 用途：独立的录制指导/分镜脚本文件（不放在“作品项目说明文档”里），便于单独提交或交给剪辑同学。

---

## 1. 目标（评委视角）

在 1–3 分钟内，用最少操作证明三件事：

1) 不是“聊天”，而是**可执行的下一步工作流**
2) 建议可解释：why/risk/rollback/verify + **RAG 引用依据（带 source）**
3) 过程可复盘：**任务流时间线**（回合制）+ 可导出回放

---

## 2. 录制前准备（10 秒）

- 服务已启动，浏览器能打开主页
- 右上角能看到执行器切换（simulate/local）
- 如果要展示 LLM：提前在「LLM设置」里配置 Token（也可反向展示“未配 Token 仍可用规则建议”）

建议：浏览器缩放 110%–125%，只录浏览器窗口。

---

## 3. 推荐分镜脚本（总时长 90–150 秒）

### 3.1 片头（10–15s）

画面：主页 + 终端 + 右侧建议/时间线区域。

口播（可直接照读）：
- “这是 Terminal Copilot，一个 Web 终端智能副驾。你输入意图或报错，它会给出可执行的下一步建议，并告诉你为什么、风险、回滚和如何验证，整个过程沉淀到右侧时间线。”

### 3.2 主场景 1：端口占用排查闭环（40–60s）

输入（任选其一）：
- `端口 8000 被占用吗？怎么查 PID 并结束？`
- 或先执行一个会失败的启动命令（更真实）

操作：
1) 观察右侧出现建议卡（最好包含跨平台命令差异）
2) 点击“执行”其中一条建议
3) 观察终端输出 + 时间线新增一个回合

口播强调（三选二即可）：
- “每条建议都带 why/risk/verify，不是只给一行命令。”
- “这里还能看到引用依据和 source，可追溯。”
- “右侧时间线按回合记录 Plan/Execute/Verify/Next，可复盘。”

### 3.3 主场景 2：依赖缺失修复（30–45s）

输入（任选其一）：
- `No module named uvicorn`
- 或运行 `python -m uvicorn ...`（故意失败也可以）

操作：
1) 点击“执行”安装依赖建议
2) 再执行启动建议

口播：
- “从错误到修复到验证是闭环，减少搜索和试错。”

### 3.4 加分亮点（10–20s，可选）

- 演示执行器切换：simulate → local（强调“演示可控、安全”）
- 演示二次确认卡：对 warn 风险命令需要确认（注意别做破坏性操作）
- 点击“导出回放”（强调“可交接、可回看证据”）

### 3.5 收尾（5–10s）

口播：
- “核心是把终端操作变成可执行、可验证、可回放的任务流，降低门槛并提升效率。”

---

## 4. Windows 录屏方式（最省事）

### 4.1 Xbox Game Bar（推荐，0 成本）

- 开始/停止录制：`Win + Alt + R`
- 打开控制台：`Win + G`

### 4.2 OBS（更专业）

- 选择“窗口捕获”锁定浏览器
- 1080p、30fps 即可
